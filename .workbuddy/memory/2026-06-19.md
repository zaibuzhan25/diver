## 用户端黑夜模式全面修复
- **问题**：用户端切换黑夜模式后背景/头部/卡片仍为白色，完全未进入深色
- **根因**：`user-fix.css` 作为最后加载的样式文件，所有规则都硬编码了浅色值（`rgba(255,255,255,0.85)`, `#F7FAFC` 等）并带 `!important`，覆盖了 `user.html` 内联 `<style>` 中的 `[data-theme="dark"]` CSS 变量
- **修复**：重写 `user-fix.css`，为每个浅色规则补上对应的 `[data-theme="dark"]` 深色覆盖，包括：
  - body 背景、header 头部、公告横幅、Tab 导航栏
  - 计划卡片、目的地卡片、轮播区、热门方案卡片
  - 按钮（primary/secondary）、模态框、表单元素
  - 个人中心全部组件、表单区域、行程编辑器
  - 向导步骤、搜索结果、模板卡片
  - JS 动态生成元素的深色兜底（kbd、#itinerary-editor、#favorite-btn 等）
- **关键经验**：CSS 加载顺序中最后加载的 `!important` 规则会覆盖前面的所有主题变量，必须在同一个文件内提供完整的深浅双套规则

## 方案卡片"查看详情"按钮流光霓虹边框
- **需求**：用户要求给方案卡片上的"查看详情"按钮加一个会随字体颜色变化的动态外框
- **实现**：
  - 用 `::before` 伪元素 + `mask-composite: exclude` 技术创建渐变边框（非 border 属性）
  - 渐变色从 `coral → coral-light → coral-dark → gold` 循环，3 秒一圈线性流动
  - 默认 70% 透明度低调显示；hover 时 100% + 外发光 `box-shadow` + 文字发光 `text-shadow`
  - 深色模式下边框不透明度提升到 85%，带微弱内发光增强可见性
  - 按钮 hover 时上浮 1px + 文字加深色 + 发光效果
- **技术要点**：使用 `-webkit-mask-composite: xor` 实现镂空边框效果，兼容 Safari/Chrome/Firefox
- **修改文件**：`user.html`（CSS 新增 `.plan-card .btn-outline::before` + `@keyframes btnGlowFlow`）

## 原生日期选择器弹窗深色模式适配
- **问题**：点击日期输入框后弹出的浏览器原生日历（calendar picker）在深色模式下仍为白底，与页面风格不协调
- **根因**：原生日期选择器是系统级 UI，不受普通 CSS 影响；需要通过 `color-scheme` CSS 属性提示浏览器使用深色主题渲染

## 点击涟漪特效
- **需求**：用户要求在用户端页面加入点击特效
- **实现**：珊瑚色（#E8734A）涟漪扩散效果，点击按钮/卡片/标签页时从鼠标位置扩散出半透明圆形波纹，0.7s 后消失
- **CSS**：`.ripple-blob` + `@keyframes rippleSpread`（cubic-bezier 缓动）
- **JS**：`initRipple()` 函数，全局 `click` 事件委托，`closest()` 匹配交互元素，动态创建/移除 `.ripple-blob` 节点
- **覆盖元素**：所有按钮、卡片（方案/目的地/统计/收藏/历史）、个人中心组件、向导步骤、行程编辑器按钮
- **初始化**：在 `NebulaInteractions.init()` 之后调用 `initRipple()`
- **缓存版本**：更新至 `v=14`

- JS 涟漪效果（`.ripple-blob` / `.ripple2-circle`）多次尝试均未生效（可能被 Vite HMR 缓存或外部 CSS 覆盖干扰）
